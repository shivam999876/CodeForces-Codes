Simons and Making It Beautiful</div><div class="time-limit"><div class="property-title">time limit per test</div>1 second</div><div class="memory-limit"><div class="property-title">memory limit per test</div>256 megabytes</div><div class="input-file input-standard"><div class="property-title">input</div>standard input</div><div class="output-file output-standard"><div class="property-title">output</div>standard output</div></div><div><p> </p><div class="epigraph"><div class="epigraph-text"><span class="tex-font-style-it">Those cling-on troubles that ride my stride — cough up the fee, or step aside.</span></div><div class="epigraph-source">— SHUN, <span class="tex-font-style-it"><a href="https://open.spotify.com/track/4hpmSSMqgesMBCc3GNopCF">TAKE IT AWAY</a></span></div></div><p>For a permutation$$$^{\text{∗}}$$$ $$$r$$$ of length $$$m$$$, we call an index $$$i$$$ ($$$1\le i\le m$$$) <span class="tex-font-style-it">ugly</span> if and only if $$$i=\max(\{r_1,r_2,\ldots,r_i\})$$$.</p><p>Simons has a permutation $$$p$$$ of length $$$n$$$, and he can perform the following operation on $$$p$$$ <span class="tex-font-style-bf">at most once</span>:</p><ul> <li> Choose two indices $$$i$$$ and $$$j$$$ ($$$1\le i\ne j\le n$$$), then swap $$$p_i$$$ and $$$p_j$$$. </li></ul><p>Find a permutation $$$q$$$ that can be obtained from $$$p$$$ by performing the above operation at most once, such that the number of <span class="tex-font-style-it">ugly</span> indices in $$$q$$$ is minimized.</p><div class="statement-footnote"><p>$$$^{\text{∗}}$$$A permutation of length $$$n$$$ is an array consisting of $$$n$$$ distinct integers from $$$1$$$ to $$$n$$$ in arbitrary order. For example, $$$[2,3,1,5,4]$$$ is a permutation, but $$$[1,2,2]$$$ is not a permutation ($$$2$$$ appears twice in the array), and $$$[1,3,4]$$$ is also not a permutation ($$$n=3$$$ but there is $$$4$$$ in the array). </p></div></div><div class="input-specification"><div class="section-title">Input</div><p>Each test contains multiple test cases. The first line contains the number of test cases $$$t$$$ ($$$1 \le t \le 100$$$). The description of the test cases follows. </p><p>The first line contains an integer $$$n$$$ ($$$1\le n\le 500$$$) — the length of $$$p$$$.</p><p>The second line contains $$$n$$$ integers $$$p_1,p_2,\ldots,p_n$$$ ($$$1\le p_i\le n$$$, all $$$p_i$$$-s are distinct) — the elements of $$$p$$$.</p></div><div class="output-specification"><div class="section-title">Output</div><p>For each test case, print $$$n$$$ integers $$$q_1,q_2,\ldots,q_n$$$ — the permutation you found.</p><p>If there are multiple possible permutations, you may output any.</p></div><div class="sample-tests"><div
